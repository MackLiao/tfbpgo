## Headline assessment

**The 2026-05-21 audit is stale; the rewrite is now a genuinely faithful port of the core science.** The big-ticket P0 claims from the old audit are closed: the correlation SQL (Pearson + Spearman) is a near-line-for-line port of Shiny's `_corr_pair_sql_impl`; the comparison plot is a real faceted boxplot (not the old heatmap) and the hackett perturbation-filter-skip matches `queries.py:432`; the schema_version=4 manifest plumbing is complete; Home/Binding/Perturbation/Comparison/Select all exist and structurally match.

**But the rewrite is not at parity.** Two genuine P0 blockers (one reproduced live on real data) and a cluster of P1 divergences remain — and notably, the **most consequential ones were missed by both prior passes** because they only manifest against the *real* artifact (where `{db}`/`{db}_meta` share metadata columns, where regulator sets exceed 1000, where zero-variance groups exist) or on interaction paths no test exercises. The synthetic fixture is the through-line: it masked the first-load matrix 400 (P0-3) entirely, since its `{db}`/`{db}_meta` tables don't overlap the way labretriever's materialized tables do.

### The two verified P0 blockers (see [CRITICAL.md](CRITICAL.md))

Both crash the Select-Datasets intersection matrix with the same HTTP 400 (`CheckField` rejecting a filter field), but have different triggers and fix sites:

1. **P0-3 — default filters 400 the matrix on first load (reproduced live on real data).** `field_manifest` drops the experimental-condition columns (`condition`/`time`/`treatment`) because `write_field_manifest` treats every column shared by `{db}` and `{db}_meta` as a join key and excludes it (`manifests.py:355`) — but the seeded `default_filters` reference exactly those fields, so the matrix 400s on the default selection with zero user action. Fixture-masked; only surfaced once the real artifact was built.
2. **P0-2 — common-regulators flow 400s end-to-end.** Clicking an off-diagonal cell → "Select N common regulators" writes a `regulator_locus_tag` filter to every active dataset, but that field is hidden from `field_manifest` and the matrix/breakdown/export handlers reject unknown filter fields (only `/corr` strips it). *Orchestrator-verified against HEAD.*

> **Withdrawn:** the workflow's other claimed P0 — *NaN correlation → 500 for the whole correlations page* — was **refuted on direct inspection**. The guard the agents reported missing is present and correct at `binding_corr.go:243` (`if math.IsNaN(...) || math.IsInf(...) { continue }`), explicitly mirroring Shiny's `df.dropna`. Do not chase it. Details in [CRITICAL.md](CRITICAL.md).

## Cross-cutting root cause: the test fixture

Most of the P0/P1s were invisible to CI because the fixture was **one dataset per data_type**, so binding/perturbation/comparison correlations collapsed to empty pair-sets and never produced a multi-sample group, a >1000-regulator intersection, or a real off-diagonal matrix cell. **`9cbc3ae` already widened the binding side** — `harbison` is now a second active binding dataset with an IEEE-NaN cell, a zero-variance column, and a reserved-keyword column, which is what let it pin the NaN class and reserved-keyword paths.

**Remaining fixture work, in priority order:** add a **second perturbation dataset** (un-masks perturbation multi-pair + the corr boxplot across pairs), ensure a **hackett non-analysis-set row** survives into a query (surfaces SQL-1), and add a backend test that applies a **`regulator_locus_tag` filter** to `/selection/matrix` (surfaces the P0). With 2 binding datasets now present, a callingcards×harbison off-diagonal matrix cell + common-regulators flow is testable today.
